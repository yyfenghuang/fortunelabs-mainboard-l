Kenapa Arsitektur Ini Lolos Uji "Matikan Router"?Mari kita bedah perilakunya saat kamu melakukan simulasi mematikan Wi-Fi router:Deteksi Instan: Begitu router mati, chip radio ESP32 kehilangan beacon pemancar dan tumpukan driver internal memicu event WIFI_EVENT_STA_DISCONNECTED.Isolasi State: Di dalam handler, bendera s_wifi_connected dan s_mqtt_connected langsung diturunkan menjadi false.Task Tetap Selamat: task_network_telemetry yang sedang berjalan di latar belakang akan mendeteksi perubahan bendera ini melalui mutex. Dia tidak akan memanggil fungsi esp_mqtt_client_publish yang bersifat blocking network, melainkan memilih melompati siklus kirim (skipping tick). Alhasil, task tidak akan hang dan tidak memicu Watchdog Timer (TWDT).Looping Mandiri Jaringan: Sembari software utamamu aman berjalan, baris esp_wifi_connect() di dalam event handler akan terus berupaya mengetuk pintu router secara asinkron setiap kali proses gagal.Auto-Recovery Penuh: Begitu router kamu nyalakan kembali, jabat tangan (handshake) Wi-Fi terjalin $\rightarrow$ IP_EVENT_STA_GOT_IP aktif $\rightarrow$ fungsi esp_mqtt_client_start() dipanggil ulang $\rightarrow$ Broker terkoneksi $\rightarrow$ Telemetri kembali mengalir mulus. Semua terjadi di balik layar tanpa intervensi esp_restart() (reboot).
